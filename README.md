# 프로젝트 소개 
* 퍼피독 'https://puppydog.co.kr' 사이트를 클론 코딩 하였습니다
* JDBC를 활용하여 데이터베이스와 연동 하였습니다

# 세부내용 
* 유저: 회원 가입, 회원 탈퇴,게시글 조회, 정보수정, 장바구니,  
* 관리자: 게시물 작성, 게시글 삭제, 회원 조회 

# 문제점 

### 소통의 문제점
* 팀원들 간의 소통이 원활하지 않아 프로젝트 진행에 어려움이 있었습니다
* 정보 전달이 늦어지거나 일관성이 부족 하였습니다 

### 팀원간의 역할
* 백엔드와 프론트를 명확하게 정하지 않아 각자의 역할이 뚜렷하지 않았습니다
* 이로 인해 팀원들 간의 협업이 어려워졌고 시간 낭비가 발생 했습니다 

### 복잡한 폴더 구조
* 프로젝트의 폴더 구조가 너무 복잡하였고 혼란스러웠습니다 이로 인해 개발 및 유지보수가 어려웠습니다 

### 관리자 로그인시 세션이 사라지는 현상
* 관리자가 로그인한 후에도 세션이 지속되지 않고 사라지는 문제가 발생했습니다  이로 인인해 게시글 기능을 수행할 수 없는 문제가 발생했습니다 

### 관계형 데이터베이스에 대한 지식 부족
* ERD설계가 늦어 지게 되었습니다 또한 데이터베이스 설계 단계에서 발생한 오류로 인해 프로젝트 일정이 지연되었습니다. 

# 후기 
* 이번 프로젝트를 진행하면서 팀원 간의 역할 분담과 소통이 핵심 요소임을 명확히 이해하게 되었습니다 
* 협업 도구의 필요성을 느끼게 되었습니다 또한 폴더 구조와 아키텍처 설계의 중요성을 깨달습니다 
* DB 기초 지식을 보충해야 한다는 필요성을 느끼게 되었습니다.
